Workflow Rules Spec v1

Deterministic operational rules for intake, matching, extraction, classification, and admin operation.

Core Rules

  1. Only active channels are synced.
  2. Message dedupe is by message_id.
  3. Match priority: shipment number, thread/reference, body references, doc refs, then create.
  4. If doc-ref match is ambiguous, reuse latest candidate and flag ambiguity.
  5. Conservative status mode defaults ON for request/future wording.
  6. Derived board status supports In Transit for picked-up not-delivered loads.
  7. Lane target format is City, ST -> City, ST.
  8. Generic customer labels are blocked; customer must be company-like proper noun.
  9. Noise emails without shipment intent should be skipped.

Admin Rules

  • Keep one active channel per source mailbox/folder.
  • Delete duplicate channels to reduce duplicate pulls.
  • Use query windows intentionally (2d for strict ops, 7d for catch-up).
  • Use vocabulary for phrase mapping only, not core matching logic.
  • If shipment-critical data is ambiguous, populate Comments on board.
