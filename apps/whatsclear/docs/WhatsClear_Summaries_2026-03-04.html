WhatsClear Summaries | Generated March 4, 2026

1) Non-Technical Workflow Summary

What WhatsClear does today: WhatsClear ingests shipment emails and digital PDFs, updates shipment records, flags uncertain cases for manual review, and keeps the master tracking sheet updated.

Business Flow

  1. Inbound shipment emails arrive in configured channels.
  2. WhatsClear ingests message content and PDF attachments.
  3. It matches to an existing shipment (shipment #, thread/reference, or doc references) or creates a new record.
  4. It classifies status and applies lifecycle/handoff checks.
  5. If uncertain or conflicting, it marks manual verification required.
  6. Operations resolves flagged records in the web UI.
  7. Updated shipment data is synced to the master sheet output.

Automation vs Human Work

Automated

  • Duplicate message prevention
  • Best-effort shipment matching
  • Status classification from known phrases
  • Digital PDF extraction
  • Audit/KPI logging

Human Required

  • Enter shipment number when missing
  • Resolve manual verification flags
  • Correct conflicts/edge cases
  • Manual overrides with reason

Current Limits

  • No auto-generated shipment numbers.
  • No OCR for scanned/image PDFs.
  • Verification queue is simplified to manual-flagged shipments.
  • Microsoft 365 ingestion is not implemented in the service logic.
  • Realtime is currently polling-based behavior.

2) Admin SOP (Current Build)

Initial Setup

  1. Start the WhatsClear web server.
  2. Login as admin.
  3. Create intake channel(s): provider, credentials/query, folder, polling interval, active=true.
  4. Run test/sync and confirm successful sync timestamp.

Daily Checklist

  1. Check channel health (active, last sync, last error).
  2. Trigger manual sync when needed.
  3. Review verification backlog with operations.
  4. Review KPIs and audit activity for anomalies.

Runbook

Sync Error

  1. Inspect channel last_error.
  2. Validate credentials/host/folder.
  3. Re-run sync and confirm last_successful_sync updates.

High Verification Backlog

  1. Prioritize high-impact shipments.
  2. Add vocabulary for recurring unmatched wording.
  3. Coordinate with ops on missing data/document patterns.

Governance Caveats

  • Current users/auth model is MVP-level.
  • Credential handling is not production-hardened secret management yet.
  • Retry/ignore intake errors adjust status records but do not fully replay original failed payloads.
